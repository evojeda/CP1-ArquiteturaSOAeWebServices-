### 📌 Checkpoint 1 – Web Service SOAP (Server + Client)

Este projeto foi desenvolvido com o objetivo de aplicar conceitos de **Arquitetura Orientada a Serviços (SOA)** e desenvolvimento de **Web Services SOAP**, contemplando tanto o lado **Server (publicação)** quanto o lado **Client (consumo)**.

## 🎯 Objetivo do Projeto

Desenvolver um sistema baseado em serviços que simula uma vinícola, permitindo:

* Publicação de serviços SOAP
* Consumo de serviços via cliente
* Comunicação entre sistemas distribuídos

---

## 🧩 Arquitetura do Projeto

O sistema foi dividido em dois lados:

- 🔹 Server (WinerySys)
Responsável por criar e publicar os serviços SOAP.

- 🔹 Client (WineOrderClient)
Responsável por consumir os serviços publicados.

---

## Serviços Desenvolvidos

## 1. WineStockService

Responsável por:
* Exibir menu de vinhos
* Realizar pedidos

## 2. WineWarningService

Responsável por:
* Enviar alertas de estoque
